3. DESCRIPCIÓN DEL SISTEMA FÁCTICO

3.1. Características Generales

Fáctico es una aplicación web (SaaS - Software as a Service) de facturación electrónica y contabilidad básica, diseñada específicamente para autónomos y pequeñas empresas en España. El sistema integra:

  • Generación de facturas emitidas y registro de facturas recibidas
  • Facturación electrónica (PDF, FacturaE/UBL, FACe para Administraciones Públicas)
  • Contabilidad automática conforme al Plan General Contable (PGC)
  • Gestión de cobros y pagos, remesas SEPA, conciliación bancaria
  • Generación de modelos tributarios (303, 347, 111, 131, 349)
  • Integración con Inteligencia Artificial para automatización de procesos
  • Sistema multiusuario con control de roles y permisos
  • Cumplimiento total con el Real Decreto 1007/2023 (VERI*FACTU)

3.2. Arquitectura Técnica

  • Frontend: Progressive Web App (PWA) desarrollada en React
  • Backend: API RESTful desarrollada en Python (FastAPI)
  • Base de datos: PostgreSQL con cifrado de datos sensibles
  • Infraestructura: Servicios cloud con alta disponibilidad
  • Comunicaciones: HTTPS (TLS 1.2+) y mTLS para Verifactu

4. CUMPLIMIENTO DEL REAL DECRETO 1007/2023

Fáctico cumple con TODOS los requisitos establecidos en el RD 1007/2023:

4.1. Integridad de Registros de Facturación (Art. 6)

  • ✅ Cada registro de facturación contiene todos los campos obligatorios establecidos en el Anexo I
  • ✅ Los registros son inmutables una vez generados (no se pueden modificar sin dejar trazabilidad)
  • ✅ Se generan registros de alta, anulación y rectificación según corresponda
  • ✅ Los datos fiscales del emisor y destinatario se conservan tal como estaban en el momento de emisión

4.2. Conservación y Trazabilidad (Art. 7)

  • ✅ Todos los registros de facturación se conservan durante el plazo legal mínimo de 6 años
  • ✅ Los registros se almacenan en formato electrónico, ordenados cronológicamente
  • ✅ Se garantiza la trazabilidad completa de todas las operaciones mediante auditoría contable
  • ✅ Los registros pueden ser exportados en formato legible (XML, CSV, PDF) en cualquier momento

4.3. Encadenamiento de Registros (Art. 8)

Fáctico implementa un sistema de encadenamiento criptográfico de registros de facturación:

  • ✅ Cada registro contiene la huella digital (hash SHA-256) del registro anterior
  • ✅ Se utiliza el algoritmo HMAC-SHA-256 con clave secreta única por empresa
  • ✅ El primer registro de cada serie contiene un hash de inicialización (genesis)
  • ✅ Cualquier intento de modificación o eliminación de un registro intermedio rompe la cadena, lo que permite detectar manipulaciones
  • ✅ La integridad de la cadena se verifica automáticamente antes de cada envío a Verifactu

Fórmula de encadenamiento implementada:

HashN = HMAC-SHA256(Clave, DatosN + HashN-1)

4.4. Remisión a la AEAT (Art. 9 y 10)

  • ✅ Fáctico envía automáticamente los registros de facturación a la AEAT mediante el sistema Verifactu
  • ✅ Comunicación segura mediante mTLS (Mutual TLS) con certificado digital
  • ✅ Generación de códigos QR con código seguro de verificación (CSV) para validación pública
  • ✅ Se registran los estados de envío (pendiente, enviado, aceptado, error) con fecha y hora exactas
  • ✅ Reintento automático en caso de fallo temporal de comunicación

4.5. Prohibición de Funcionalidades No Conformes (Art. 11)

  • ✅ NO se permite la eliminación de registros de facturación (solo anulación mediante registro de anulación)
  • ✅ NO se permite la modificación de registros de facturación ya generados (solo rectificación mediante registro de rectificación)
  • ✅ NO existe ninguna funcionalidad de "modo fantasma", "doble facturación" o similar
  • ✅ NO se permite alterar el orden cronológico de los registros
  • ✅ NO se permite la manipulación del encadenamiento de huellas digitales

⚠️ IMPORTANTE: Fáctico está diseñado específicamente para cumplir con la normativa fiscal española. Cualquier intento de uso fraudulento por parte del usuario es responsabilidad exclusiva del mismo, no del sistema.

4.6. Accesibilidad y Auditoría

  • ✅ La AEAT puede acceder en cualquier momento a los registros de facturación
  • ✅ Fáctico genera informes de auditoría con todas las operaciones realizadas
  • ✅ Los usuarios pueden exportar su contabilidad completa en cualquier momento
  • ✅ Se conservan logs de acceso y modificaciones para fines de auditoría

7. RESPONSABILIDAD DEL USUARIO

El sistema Fáctico proporciona las herramientas técnicas necesarias para el cumplimiento del RD 1007/2023, pero la responsabilidad final del correcto uso y cumplimiento fiscal recae en el usuario (empresario o profesional).

El usuario es responsable de:

  • Introducir datos fiscales y de facturación correctos y veraces
  • Verificar que los registros generados reflejen correctamente sus operaciones
  • Presentar las declaraciones tributarias correspondientes en plazo
  • Conservar documentación de respaldo (contratos, albaranes, etc.)
  • Consultar con un asesor fiscal ante cualquier duda normativa

Recomendación: Fáctico proporciona herramientas de ayuda automática, pero NO sustituye el asesoramiento de un profesional contable o fiscal. Se recomienda consultar con un gestor antes de presentar declaraciones tributarias.
